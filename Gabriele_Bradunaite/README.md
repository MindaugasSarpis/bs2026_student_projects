## What I Did

I analysed a sample investment portfolio containing 533 holdings using Python and the pandas library. The data came from a CSV file with information about each investment - including its sector, price, country, and ESG (Environmental, Social, Governance) scores. Using Python, I wrote a Jupyter notebook that loads the data, processes it, and creates visualisations. All charts and visuals were written and customised by me. The website that displays the charts together was generated using an AI prompt, purely for the sake of having a clean visual output in one place.



## Why I Made These Choices

I deliberately tried multiple chart types for the same data - vertical bar, horizontal bar, pie, lollipop, and scatter -  to show that the same information can be communicated in different ways, and that choosing the right chart is part of the analytical process. For example, I found that a horizontal bar chart works better than a vertical one when sector names are long, because the labels don't overlap. That kind of decision - noticing a problem and fixing it - is something I learned through doing.


## What I Learned

Before this course I had never opened a terminal, never written a line of code, and had no idea how Python was used.

The most important thing I learned was how to actually write small code. I now understand what a library is and why we import it at the top of the script. I understand what a DataFrame is. I also learned how to customise charts - changing colours, adjusting bar widths, adding titles and value labels — which gave me a sense of how much control you have over how information is presented visually.

The most important shift over this project was moving from copying code without understanding it, to being able to read a line and explain what it does.
